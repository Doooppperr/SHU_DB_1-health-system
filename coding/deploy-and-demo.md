# 部署与答辩演示脚本

## 1. 一键启动脚本（项目根目录）

### 开发模式

```powershell
.\scripts\start-full-dev.ps1
```

- 后端：`http://127.0.0.1:5000`
- 前端：`http://127.0.0.1:5173`

### 生产演示模式

```powershell
.\scripts\start-full-prod.ps1
```

- 后端（Waitress）：`http://127.0.0.1:5000`
- 前端预览：`http://127.0.0.1:4173`

说明：
- 如需单独启动后端：`.\scripts\start-backend-prod.ps1`
- 如需单独启动前端：`.\scripts\start-frontend-prod.ps1`

## 2. 演示账号准备

- 管理员（默认种子）：
  - 用户名：`admin`
  - 密码：`admin123`
- 普通用户：现场注册两个账号，例如：
  - `demo_user_a`
  - `demo_user_b`

## 3. 12分钟答辩演示流程（建议）

### 第1段：基础闭环（2分钟）

1. 用 `demo_user_a` 登录。
2. 进入机构列表 -> 机构详情，展示套餐数据。
3. 进入档案列表，新建1份个人档案并录入2-3个指标。

验收点：
- 已登录可访问业务页面。
- 机构/套餐和个人档案链路正常。

### 第2段：OCR闭环（2分钟）

1. 进入 `OCR上传` 页面，上传报告。
2. 展示“候选映射确认表”“未匹配字段”“过滤字段”。
3. 在候选映射表中可改选指标或忽略误匹配项。
4. 点击“确认入档”，跳转档案详情。

验收点：
- OCR上传解析成功。
- 状态可从 `parsed` 到 `confirmed`，且确认后指标才真正入档。

### 第3段：亲友授权闭环（3分钟）

1. `demo_user_a` 在亲友管理中添加 `demo_user_b`。
2. 切换 `demo_user_b` 登录，在“授权给我的关系”中打开授权。
3. 切回 `demo_user_a`，创建一份归属人为 `demo_user_b` 的档案（代传）。

验收点：
- 未授权时禁止代传，授权后允许代传。
- 档案归属人与上传人可区分显示。

### 第4段：趋势分析闭环（2分钟）

1. 进入“指标趋势”页面。
2. 选择归属人+指标，展示折线图与参考区间线。
3. 展示最小值/最大值/最新值统计。

验收点：
- 趋势按“归属人+指标”聚合。
- 支持本人和已授权亲友趋势查询。

### 第5段：评论审核闭环（3分钟）

1. `demo_user_a` 在机构详情提交评论（先确保有该机构上传记录）。
2. 用 `admin` 登录进入“评论审核”，将评论设为可见。
3. 回到机构详情，展示公开评论已出现。

验收点：
- 用户提交评论需满足“有该机构体检记录”门槛。
- 管理员可审核控制 `is_visible`。

## 4. 常见问题排查

1. 前端无法访问后端：确认后端监听 `5000`，并检查 Vite 代理配置。
2. 评论无法提交：先确认该用户上传过该机构档案。
3. 无法进入审核页：确认登录的是 `admin` 角色账号。
4. OCR映射少：样本可能不是指标页，可用手动录入补充演示。

## 5. OCR映射为0时的现场处理（新增）

当出现 `mapped_count=0`、`unmatched_count` 较高时，建议按以下话术和操作演示：

1. 先说明原因：
  - 该报告页可能包含机构信息、建议文本、参考范围碎片，非纯指标页。
2. 展示系统容错：
  - 页面会给出“未匹配字段”和“过滤字段”，不会误写入档案。
3. 使用候选映射确认表：
  - 若有候选项，手动改选后点击确认入档。
4. 无候选项时兜底：
  - 跳转档案详情页手动补录关键指标（FBG/TC/LDL/ALT/UA等），继续演示趋势与后续流程。

建议答辩话术：
- “系统在低质量或非指标页下优先保证正确性，先给出可审阅候选，再由用户确认入档，避免错误自动入库。”
