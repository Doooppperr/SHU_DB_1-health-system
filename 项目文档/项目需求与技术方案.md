# 体检评价与健康档案系统：项目需求与技术方案

## 1. 项目目标

构建一个可以在个人电脑上直接运行的 B/S 健康管理平台。用户通过浏览器管理个人和亲友体检档案，所有业务数据保存在项目本地 SQLite 数据库中。

## 2. 角色

- 访客：无需登录即可浏览登录/注册页，并使用 AI 查询公开系统功能。
- 普通用户：管理个人资料、体检档案、亲友关系、授权和评论。
- 管理员：管理用户、审核评论，并拥有普通用户能力。

## 3. 功能需求

### 3.1 用户认证

- 注册、登录和退出。
- 图片验证码。
- JWT Access Token 与 Refresh Token。
- 用户查看和修改自己的资料。

### 3.2 机构与套餐

- 浏览体检机构列表和详情。
- 查看机构提供的体检套餐。
- 用户上传过该机构档案后才允许评论。

### 3.3 健康档案

- 创建、查看和删除档案。
- 区分档案归属人和上传人。
- 记录体检机构、套餐、体检日期和报告文件。
- 手动添加、修改和删除健康指标。
- 根据参考范围自动判断数值是否异常。

### 3.4 OCR

- 支持图片和 PDF。
- 支持本地 Mock 与真实 OCR API 两种模式。
- 解析结果先进入候选确认页。
- 用户确认映射后才正式写入健康指标。

### 3.5 亲友授权

- 添加、改名和删除亲友关系。
- 被添加方控制授权状态。
- 授权后允许对方代传档案和查看趋势。

### 3.6 趋势分析

- 按档案归属人和指标查询时间序列。
- 展示折线图、参考区间、最新值、最小值和最大值。

### 3.7 AI 智能助手

- 登录前通过可拖动悬浮球提供注册、登录和公开系统功能导览。
- 登录后支持指标含义、基础健康问题和一般生活建议，但不诊断、不推荐处方药。
- 用户可显式选择同一归属人的最多 5 份本人或已授权亲友档案；详情页默认选择当前档案。
- 普通问题由 DeepSeek V4 Flash 回答，复杂问题转人工客服，疑似急症直接提示拨打 120。
- 对话仅保留最近 10 轮；更早内容合并为摘要。浏览器刷新后恢复，结束对话或退出登录后清空，后端数据库不保存聊天记录。
- 选择健康档案后必须确认指标将发送至 DeepSeek API 处理。

### 3.8 评论和后台管理

- 用户提交评分与评论。
- 用户查看和删除自己的评论。
- 管理员控制评论是否公开。
- 管理员查看、修改和删除用户。

## 4. 技术方案

| 层级 | 选型 |
|---|---|
| 浏览器端 | Vue 3 + Vite + Element Plus |
| 状态与路由 | Pinia + Vue Router |
| 网络请求 | Axios |
| 图表 | ECharts |
| AI | DeepSeek V4 Flash、固定 FAQ、安全分流 |
| 服务端 | Flask |
| ORM | Flask-SQLAlchemy |
| 数据库 | SQLite |
| 鉴权 | Flask-JWT-Extended |
| 本机演示服务 | Waitress |
| 自动化测试 | Pytest |

## 5. 本地运行架构

```text
浏览器
  │
  ▼
Vue/Vite :5173（演示模式为 :4173）
  │ /api 与 /uploads 代理
  ▼
Flask/Waitress :5050
  ├─ backend/instance/health_system.db
  └─ backend/uploads/
```

系统没有远程数据库依赖。开发模式和本机演示模式共享本地 SQLite 文件。

AI 启用真实模式时，只有 Flask 后端通过 HTTPS 调用 DeepSeek；浏览器不直接持有 API Key：

```text
浏览器 AI 侧栏
  │ POST /api/ai/chat（JWT 可选）
  ▼
Flask AI 网关
  ├─ 本地 FAQ / 急症规则
  ├─ SQLite 只读档案权限校验
  └─ DeepSeek V4 Flash（普通问答与 JSON 安全分类）
```

### 5.1 AI 接口约定

- 输入：当前问题、最多 10 轮完整历史、更早摘要、最多 5 个档案 ID。
- 输出：回答文本、安全决策、来源、更新后摘要和压缩消息数。
- `answer` 返回科普；`support` 由后端替换成人工客服模板；`emergency` 固定提示 120。
- 未登录请求不能携带档案；登录请求必须重新验证档案归属、亲友授权和确认状态。
- 选择多份档案时必须属于同一人，避免混淆不同用户的健康数据。

## 6. 数据设计原则

- 关系表达到 3NF；基于已识别函数依赖的核心表满足 BCNF。
- 使用外键表达实体关系。
- 使用唯一约束保证候选键和防止重复数据。
- 使用检查约束限制角色、状态、评分和数值范围。
- 跨表权限和业务规则由服务端事务校验。
- 测试使用内存 SQLite，与实际本地数据隔离。

## 7. 非功能需求

- Python 3.10+、Node.js 18+。
- 支持 PowerShell 一键启动。
- 数据库和上传文件可以直接复制备份。
- 密码使用 bcrypt 哈希。
- 敏感 OCR 凭据只保存在被 Git 忽略的 `backend/.env`。
- DeepSeek API Key 只从环境变量读取，不进入前端或数据库。
- AI 对话、摘要和回复不写入 SQLite；刷新恢复只使用当前标签页的 `sessionStorage`。
- 对匿名与登录 AI 请求分别限流，模型失败时返回统一错误，不向前端暴露供应商响应细节。
- 用户选择档案后必须明确确认指标将发送至第三方模型处理。
