# 本地运行与演示指南

## 1. 首次准备

```powershell
Set-Location .\backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Set-Location ..\frontend
npm install

Set-Location ..
```

复制环境变量模板：

```powershell
Copy-Item .\backend\.env.example .\backend\.env
```

本地数据库已经位于：

```text
backend/instance/health_system.db
```

不需要安装独立数据库服务。

## 2. 开发模式

在项目根目录执行：

```powershell
.\scripts\start-full-dev.ps1
```

访问 `http://127.0.0.1:5173`。

该脚本会：

1. 在新 PowerShell 窗口启动 Flask 后端。
2. 在当前窗口启动 Vite 前端。
3. 将前端 `/api` 和 `/uploads` 请求代理到 `127.0.0.1:5050`。

## 3. 本机演示模式

```powershell
.\scripts\start-full-prod.ps1
```

访问 `http://127.0.0.1:4173`。

后端由 Waitress 启动，前端使用 Vite Preview，数据库仍是本地 SQLite。

## 4. 单独启动

```powershell
.\scripts\start-backend-dev.ps1
.\scripts\start-frontend-dev.ps1
```

或：

```powershell
.\scripts\start-backend-prod.ps1
.\scripts\start-frontend-prod.ps1
```

## 5. 默认账号

```text
admin / admin123
```

## 6. 建议演示顺序

1. 未登录时打开 AI 悬浮球，询问“如何注册账号”。
2. 登录并查看机构与套餐。
3. 创建一份手动档案并添加指标。
4. 上传 OCR 报告，检查候选映射并确认入档。
5. 在档案详情页打开 AI，确认当前档案被默认选择并询问指标含义。
6. 注册第二个用户，演示亲友授权、代传和 AI 选择同一亲友的多份档案。
7. 查看指标趋势图。
8. 提交机构评论。
9. 使用管理员账号审核评论并查看用户管理。

## 7. OCR 模式

完全离线演示时，在 `backend/.env` 设置：

```env
OCR_USE_MOCK=1
```

真实 OCR 模式需要网络和有效 OCR API 凭据，但不影响数据库的本地运行方式。

## 8. AI 模式

真实 DeepSeek 模式在 `backend/.env` 设置：

```env
AI_PROVIDER=deepseek
AI_USE_MOCK=0
DEEPSEEK_API_KEY=在 DeepSeek 控制台新建的密钥
DEEPSEEK_MODEL=deepseek-v4-flash
AI_SUPPORT_PHONE=实际人工客服电话
```

真实密钥不能写入 `.env.example`、README、测试或 Git 提交。完全离线演示可改为：

```env
AI_USE_MOCK=1
```

Mock 会验证界面、权限和会话流程，但不会产生真实模型回答。固定 FAQ 在未配置模型时仍可回答常见注册和系统功能问题。

## 9. 常见问题

### 端口被占用

```powershell
.\scripts\start-full-dev.ps1 -BackendPort 5051
```

### 后端无法启动

确认虚拟环境存在：

```powershell
Test-Path .\backend\.venv\Scripts\python.exe
```

重新安装依赖：

```powershell
.\backend\.venv\Scripts\python.exe -m pip install -r .\backend\requirements.txt
```

### 数据库需要恢复

先关闭后端，然后用备份文件替换：

```text
backend/instance/health_system.db
```

### 数据库模型增加了新约束

先关闭后端，然后执行：

```powershell
Set-Location .\backend
.\.venv\Scripts\python.exe .\scripts\upgrade_local_database.py --check-only
.\.venv\Scripts\python.exe .\scripts\upgrade_local_database.py
```

升级前数据库会自动备份到 `backend/instance/`。

### 前端接口请求失败

检查后端健康接口：

```powershell
Invoke-RestMethod http://127.0.0.1:5050/api/health
```

预期结果：

```json
{"status":"ok"}
```

### AI 提示“服务尚未配置”

确认 `backend/.env` 中存在非空 `DEEPSEEK_API_KEY`，保存后重启后端。不要在终端截图、聊天记录或日志中展示密钥。

### 复杂问题没有显示电话号码

在 `backend/.env` 配置 `AI_SUPPORT_PHONE` 并重启后端。未配置时系统只会提示联系人工客服。

### AI 回复过于频繁

匿名和登录用户默认分别限制每分钟 10 次和 30 次请求，可通过 `AI_GUEST_RATE_LIMIT_PER_MINUTE` 与 `AI_AUTH_RATE_LIMIT_PER_MINUTE` 调整。
