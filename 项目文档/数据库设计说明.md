# 康康健健 HealthDoc 数据库设计说明

> 文档状态：SQLite/openGauss schema v6 当前物理设计与迁移说明，更新于 2026-07-16。

## 1. 版本与运行方式

本地数据库为 `backend/instance/health_system.db`，使用 `PRAGMA user_version=6`。新空库创建完整 v6；非空旧库的版本、表、列或命名约束不符合当前 metadata 时拒绝启动，并提示运行 `backend/scripts/upgrade_local_database.py`。

生产通过 `DATABASE_URL` 连接 openGauss。`create_all()` 只用于空库初始化；合成演示库全量同步使用 `migrate_sqlite_to_gaussdb.py --replace`，同步前必须完成离线备份。

## 2. 当前表

| 表 | 用途 | 关键约束 |
|---|---|---|
| `users` | 三类账号与普通用户健康身份 | 用户名、非空邮箱、健康身份码唯一；角色与机构/健康身份检查 |
| `friend_relations` | 亲友只读授权 | 用户对唯一，禁止自关联 |
| `institutions` | 体检机构 | 名称与分院联合唯一，软停用 |
| `packages` | 机构套餐 | 机构与名称联合唯一，价格非负，软停用 |
| `appointments` | 用户体检预约与履约状态 | 用户与有效日期联合唯一，五状态检查，套餐/用户快照 |
| `package_change_requests` | 套餐全生命周期审核 | 操作与状态检查，前后 JSON、申请人/审核人和时间留痕 |
| `institution_invites` | 每机构当前邀请码 | 机构唯一、哈希唯一、状态检查 |
| `institution_images` | 机构相册 | 存储键唯一，机构内排序唯一且为 0–7 |
| `indicator_categories` | 指标分类 | 名称唯一 |
| `indicator_dicts` | 标准指标字典 | 编码唯一，参考范围检查，自测白名单标志 |
| `self_measurements` | 本人日常测量 | 用户/指标外键，值非负，测量时间索引 |
| `institution_reports` | 机构报告及状态机 | 预约一对零或一、机构/健康身份码/日期唯一、状态检查 |
| `report_indicators` | 标准化报告指标 | 报告与字典联合唯一，值非空，来源检查 |
| `comments` | 机构评论 | 评分 1–5，内容非空 |

schema v6 不包含旧 `exam_registrations`，正式预约统一使用 `appointments`。AI 会话、AI 分析结果、验证码和 JWT 不进入业务表。

## 3. 关系

```text
institutions 1 ── N users（institution_admin.managed_institution_id）
institutions 1 ── 0..1 institution_invites
institutions 1 ── N institution_images / packages / appointments / package_change_requests / institution_reports / comments

users 1 ── N self_measurements
users 1 ── N institution_reports（matched_user，可空）
users 1 ── N institution_reports（creator，可空，账号删除后保留用户名快照）
users 1 ── N comments / friend_relations

packages 1 ── N appointments / package_change_requests
appointments 1 ── 0..1 institution_reports
institution_reports 1 ── N report_indicators
indicator_dicts 1 ── N self_measurements / report_indicators
```

## 4. 关键字段

### 4.1 `users`

- `role` 为 `user/institution_admin/admin`。
- 机构账号必须填写 `managed_institution_id` 且没有 `health_id`；普通用户必须具有唯一 `health_id` 且不能绑定机构；管理员两者均为空。
- 普通用户资料包括 `real_name/birth_date/gender/allergy_history/medical_history`。
- `is_active` 控制登录、刷新和所有受保护请求。
- `password_hash` 使用 bcrypt，不保存明文。

### 4.2 `self_measurements`

- `user_id` 删除时级联；业务层只允许本人 CRUD。
- `indicator_dict_id` 必须属于 `HEIGHT/WEIGHT/HR/TEMP/SPO2/FBG` 自测白名单。
- `value` 为非负 `Numeric(14,4)`；`measured_at` 保存实际测量时间。
- 同用户、同指标、同日允许多次测量。

### 4.3 `appointments`

- `status` 为 `unfulfilled/awaiting_report/fulfilled/invalidated/cancelled`。
- `active_date_key` 对未履约、待上传报告和已履约保留预约日期，对取消或失效清空；与 `user_id` 联合唯一，实现同一用户同一天全平台一条有效预约。
- `user_name_snapshot/user_health_id_snapshot/package_name_snapshot/package_price_snapshot` 保存预约时事实，套餐后续改名、改价或下架不影响历史。
- `cancelled_at/attended_at/invalidated_at/fulfilled_at` 记录状态转换时间。
- 名额按机构、预约日期及非取消/失效状态实时计数；机构 `daily_appointment_limit` 为空表示不限量。

### 4.4 `package_change_requests`

- `action` 为 `create/update/deactivate/reactivate`，`status` 为 `pending/approved/rejected/withdrawn`。
- `before_data/proposed_data` 保存 JSON 快照；申请人、审核人、备注与各阶段时间永久留痕。
- 同一现有套餐同一时刻只允许一条待审申请；机构可撤回，管理员通过时才在同一事务修改正式 `packages`。

### 4.5 `institution_reports`

- `institution_id` 来自登录机构账号的服务端绑定，不能由前端切换。
- `created_by_user_id` 可因机构账号删除而置空；`created_by_username_snapshot` 永久保留创建时用户名。
- `appointment_id` 对新报告唯一且必定指向待上传报告预约；schema v5 历史报告允许为空。
- `subject_name_snapshot` 与 `subject_health_id` 来自预约用户快照。
- `matched_user_id` 在成功提交时指向身份完全一致的启用普通用户；用户删除时报告级联删除。
- `status` 只允许 `draft/locked/published`。
- `ocr_diagnostics` 保存必要解析诊断；`temporary_file_url` 只在草稿期存在，锁定时清空并删除文件。
- `locked_at/submitted_at/published_at` 记录状态时间。
- 唯一约束保证同机构、同健康身份码、同体检日期只有一份报告。
- `display_id=report{id}` 是派生展示值，不落库。

### 4.6 `report_indicators`

- `(report_id, indicator_dict_id)` 唯一。
- `value` 为规范化非空字符串；`is_abnormal` 依据字典范围计算。
- `input_source` 为 `manual/ocr`；报告删除时级联。

### 4.7 `friend_relations`

- `(user_id, friend_user_id)` 唯一且禁止自关联。
- `auth_status` 只表示被添加方是否允许对方查看时间线、报告和趋势。
- 表中没有上传、编辑或代记权限字段；亲友无法修改对方数据。

## 5. 预约与报告归档事务

1. 用户在今天起 30 天内选择启用机构和套餐；服务端锁定机构行、复核每日上限并写入 `unfulfilled`。
2. 用户可取消并清空有效日期键；机构可置 `invalidated`，或确认到检改为 `awaiting_report`。
3. 机构只能从 `awaiting_report` 创建 `draft`，身份、日期和套餐由预约快照提供；录入至少一项指标后锁定。
4. 锁定后进入 `locked`，指标不可再改且临时 OCR 文件删除；提交时再次校验预约用户仍启用。
5. 同一事务设置报告 `published` 字段及预约 `fulfilled/fulfilled_at`；任何一步失败全部回滚。
6. `published` 报告永久归档；用户、亲友、趋势和 AI 均读取同一份只读报告事实。

## 6. 趋势、删除与文件边界

- 同日存在已发布机构指标时优先采用机构值；报告不含该指标时采用该日最后一次自测。
- 删除普通用户需显式确认，并级联自测、归属报告及报告指标；删除机构账号只将创建者外键置空。
- `/uploads` 仅公开数据库登记的机构图片；`reports/` 路径始终拒绝公开访问。
- `cleanup_local_runtime.py` 只处理上传目录孤儿文件，不处理报告等待状态。

## 7. schema v6 本地迁移

`upgrade_local_database.py` 校验源库完整性，在相邻临时文件创建 v6，设置 `user_version=6`，按共有表与字段完整保留 v5 业务数据并创建预约/审核新表，验证结构、行数、外键和完整性，生成 `health_system.before-schema-v6-<时间戳>-<随机串>.db` 后原子替换正式库。
