# 康迹 HealthHub 项目文档索引

> 最后核对：2026-07-13。当前发布分支为 `main`，自动化基线为后端 143 项、前端 19 个文件 109 项；Vite production build 和生产依赖审计通过。

本目录只保留仍有明确维护职责的正式文档。历史实施进度、版本节点和待重制交付物差异统一归入《开发记录与上下文归档》，避免同一事实散落在临时文件中。

## 推荐阅读顺序

1. 新成员先读根目录 [`README.md`](../README.md)，完成安装、启动和最小验证。
2. 了解产品边界与三角色设计时读 [`项目需求与技术方案.md`](项目需求与技术方案.md)。
3. 本机部署、演示、备份或排障时读 [`本地运行与演示指南.md`](本地运行与演示指南.md)。
4. 修改表、约束或迁移脚本前读 [`数据库设计说明.md`](数据库设计说明.md) 与 [`数据库规范化说明.md`](数据库规范化说明.md)。
5. 修改 AI、SSE 或 OCR 状态机前读 [`AI与OCR开发说明.md`](AI与OCR开发说明.md)。
6. 发布前按 [`测试报告.md`](测试报告.md) 的命令和验收矩阵回归。
7. 追溯已完成工作、历史基线和维护约定时读 [`开发记录与上下文归档.md`](开发记录与上下文归档.md)。

## 单一事实来源

- 当前测试数量和构建结论以《测试报告》为准；历史数字只在《开发记录与上下文归档》中按时间保留。
- 当前数据库结构以模型、迁移脚本和《数据库设计说明》为准；范式解释以《数据库规范化说明》为准。
- 当前 AI/OCR 协议以服务端路由、前端 API 模块和《AI与OCR开发说明》为准。
- 当前启动命令以根 README 和《本地运行与演示指南》为准，不从历史记录复制旧命令。
- 密码、邀请码明文、API Key、真实健康数据、本地数据库和上传文件不得写入文档或 Git。

## 常用变更定位

| 变更类型 | 先检查的实现 | 必须同步的文档与验证 |
|---|---|---|
| 登录、注册、验证码、角色跳转 | `backend/app/auth/`、前端认证 Store 与路由 | 前后端 README、需求方案、认证/路由测试 |
| 健康档案、指标和趋势 | records/trends/indicator 服务与对应视图 | 需求方案、数据库说明、测试报告 |
| 机构、套餐、邀请码和相册 | admin/org/institutions 路由与服务 | 需求方案、演示指南、数据库说明、角色测试 |
| AI 流式对话或分析 | `backend/app/ai/`、AI Store 与助手组件 | AI/OCR 说明、Mock、SSE 与前端交互测试 |
| OCR 解析或确认 | OCR 服务、records 路由与上传视图 | AI/OCR 说明、解析/并发测试、演示指南 |
| 主题、关怀和响应式布局 | appearance Store、全局样式与工作台布局 | 前端 README、视觉/组件测试、测试报告 |
| 构建、分包或依赖 | Vite 配置、依赖清单和共享运行时 | 前端 README、测试报告、CI |
| schema 或文件存储 | models、迁移脚本、storage/record_files 服务 | 两份数据库说明、备份恢复、迁移测试 |

## 修改与发布检查

- 需求或角色变化：同步需求方案、前后端 README、路由/接口测试和开发记录。
- 数据模型变化：同步模型、迁移脚本、数据库两份说明、测试和备份/恢复说明。
- AI/OCR 变化：同步协议文档、前后端测试、Mock 行为、错误码和演示步骤。
- 构建或依赖变化：同步前端 README、测试报告和 CI；不得仅通过调高告警阈值隐藏问题。
- 每次发布至少执行后端 Pytest、前端 Vitest、Vite build、生产依赖审计和 `git diff --check`。
